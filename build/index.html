    Welcome to Ride-My-Way API

    Format of the response gotten when you hit the root of the app

    {
    "message": "Welcome to Ride-My-Way app! Your one stop place to get rides to your desired destination at reasonable prices",
    "endpoints": {
        "signup": "POST /api/v1/auth/signup",
        "login": "POST /api/v1/auth/login",
        "getAllRideOffer": "GET /api/v1/rides",
        "getOneRideOffer": "GET /api/v1/rides/:rideId",
        "makeRequestForRide": "POST /api/v1/rides/:rideId/requests",
        "createRideOffer": "POST /api/v1/users/rides",
        "getAllRequestsForRide": "GET /api/v1/GET /users/rides/:rideId/requests",
        "acceptRejectRequests": "PUT /api/v1//users/rides/:rideId/requests/:requestId"
    },
    "success": true
}

    Welcome to the Ride-My-Way API documentation! This is a carpooling application that provides drivers with the ability to create ride offers and passengers to join available ride offers. You can access the Ride-My-Way API endpoints, which allow you to signup as a new user, to login, to create a ride offer, to get information on various various ride offers available, to make a request for a ride offer, and you can also accept or reject a ride offer.

    The API was built using Javascript! You can view code examples in the dark area to the right.

    Request

    • Welcome endpoint : GET /api/v1/
    • Body (application/json)

    Response

    • Status: 200 OK
    • Message: Welcome to Ride-My-Way app! Your one stop place to get rides to your desired destination at reasonable prices
    • Body (application/json)

    HTTP Request

    POST https://ayo-ride-my-way-v1.herokuapp.com/api/v1/

    Authentication

    Ride-My-Way app needs users to be authenticated to be able to perform the actions specified above (except for viewing all available rides). On a successful sign up or sign in, a token is generated which gives users access to these privileges.

    Ride Offers

    Create a Ride offer

    This is the format of the request sent for creating a ride offer

    {
    "message": "Dave travellers",
    "destination": "Ajaokuta",
    "departurelocation": "MMA 2 terminal",
    "date": "2018-07-27"
}

    Format of the response gotten after successful ride creation

    {
    "message": "Your ride offer is created successfully",
    "data": {
        "message": "Dave travellers",
        "destination": "Ajaokuta"
    },
    "success": true
}

    This endpoint allows an authenticated user to create a ride offer.

    Request

    • Create ride endpoint: POST /api/v1/users/rides
    • Body (application/json)
    • Requires: Authentication

    Response

    • Status: 201 Created
    • Message: You are logged in!
    • Body (application/json)

    HTTP Request

    POST https://ayo-ride-my-way-v1.herokuapp.com/api/v1/users/rides

    Response codes when error occurs at ride creation

    HTTP Status code Message
    400 Missing required fields/unsupported type of input.
    409 Conflict in data.
    500 Internal server error.
    Remember — a happy ride creator is an authenticated user!

    Requests

    Request a Ride

    Format of the response gotten after successful request for a ride

    {
    "message": "Your request for this ride is registered, Pending for acceptance. Thanks a lot!",
    "success": true
}

    This endpoint allows an authenticated user to request for a ride offer.

    Request

    • Make request for ride endpoint: POST /api/v1/rides/13/requests
    • Body (application/json)
    • Requires: Authentication

    Response

    • Status: 201 Created
    • Message: Your request for this ride is registered, Pending for acceptance. Thanks a lot!
    • Body (application/json)

    HTTP Request

    POST https://ayo-ride-my-way-v1.herokuapp.com/api/v1/rides/13/requests

    Response codes when error occurs at request ride

    HTTP Status code Message
    401 validation errors
    406 Unacceptable request/request made by ride creator.
    409 Conflict in request.
    404 Unregistered ride/ ride not found.
    500 Internal server error.
    Remember — a happy requester is an authenticated user!

    This endpoint allows an authenticated user to check the requests made for a ride he/she created.

    Request

    • Get all request endpoint: GET /api/v1/users/rides/5/requests
    • Body (application/json)
    • Require: Authentication

    Response

    • Status: 200 OK
    • Message: Your ride has been requested by:
    • Body (application/json)

    HTTP Request

    POST https://ayo-ride-my-way-v1.herokuapp.com/api/v1/users/rides/5/requests

    Response codes when error occurs at getting all rides

    HTTP Status code Message
    403 Forbidden requests/ User is not the owner of the ride
    500 Internal server error.

    Accept or reject a request for a ride

    Format of the request when accepting or rejecting a request for a ride (This can only be carried out by the owner of the ride)

    {
    "status": "Accepted"
}

    Format of the response gotten after an authenticated user successfully updates the status of the request madde for his ride

    {
    "message": "The status of the request has been updated successfully",
    "success": true
}

    This endpoint allows an authenticated user to either accept or reject a request made for a ride he/she created.

    Request

    • Accept or reject request endpoint: PUT /api/v1/users/rides/3/requests/19
    • Body (application/json)
    • Require: Authentication

    Response

    • Status: 202 Accepted
    • Message: The status of the request has been updated successfully
    • Body (application/json)

    HTTP Request

    PUT https://ayo-ride-my-way-v1.herokuapp.com/api/v1/users/rides/3/requests/19

    Response codes when error occurs at getting accepting or rejecting rides requests

    HTTP Status code Message
    400 unsupported type of parameter.
    406 Unacceptable input.
    403 Forbidden requests/ User is not the owner of the ride
    500 Internal server error.

    Errors

    The Ride-My-Way API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your Token is wrong.
    403 Forbidden -- The function requested is for owners of the resource only.
    404 Not Found -- The specified ride/request could not be found.
    406 Not Acceptable -- You requested a format that isn't of the right data type.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
